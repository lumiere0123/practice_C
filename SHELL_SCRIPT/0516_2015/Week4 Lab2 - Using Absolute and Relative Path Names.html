Pathnames in Unix/Linux are divided into two types. The first is the absolute pathname, which always starts with the / (slash), commonly named the root. The second type of pathname is the relative pathname, which is aptly named, for it is relative to your current working directory location. It is important for the user to be able to create and manipulate the file structure by either using absolute pathnames or relative pathnames, or both.

First, let's take a look at a sample of an absolute pathname to create a directory off of your home directory named seneca.
Since we are using an absolute pathname to create the seneca directory, we know that the pathname must start with the /, or root. Hence, the command line:
username@matrix:~> mkdir /home/username/seneca (where "username" is your Matrix login ID).

Second, if you want to make the same directory off of your home directory using a relative pathname you would not start with the slash, or root, but would simply name the file, because your current working directory is already your home directory. Hence, the command line:
username@matrix:~> mkdir Seneca

To move around in a directory tree you can also use absolute pathnames, or relative pathnames. Using the directory tree structure shown above, let's take a look at how to navigate the entire tree. You need to replace username with your own login name.

First, your current location is your home directory which is under /home. To move to the seneca directory using an absolute pathname:
$ cd /home/username/Labs/lab4a/seneca
To do the same operation using a relative path:
$ cd Labs/lab4a/seneca

Second, your current location is library. To move to the newnham directory:
$ cd ..
This operation uses two single dots (or periods) to signify the parent directory.
(If you examine the library directory you will notice that newnham is the parent of library.)
This method of using the .. is a type of relative pathname.

Third, your current location is security. To move to the cns directory:
$ cd ../../s@y/cns
The first set of ..'s moves you to the newnham directory, the second set of ..'s moves you to the root. Then you simply declare the path down to your location.
