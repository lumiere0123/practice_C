TITLE:

LAB 2c Using Command Line FTP

OBJECTIVE:

As a result of this lab the student should be able to transfering files using command line FTP.

PART I

  1. Right click on the following link and select "Save target as..."
    vi Lab Files
  2. Save the file to the C:\TEMP folder
  3. Open a Windows Command Prompt window and make C:\ your current working directory.
  4. Start the FTP client by typing FTP and the pressing Enter. You should see a prompt that looks like this:
    ftp>
  5. You now need to open a connection to matrix.senecac.on.ca by typing:
    open matrix.senecac.on.ca
    at the ftp> prompt.
  6. Enter your matrix account name and password when prompted.
  7. Enter the command pwd and view the result. Where does it say you are? ___________________________________________
  8. Enter the command ls and view the result. Where do you think you are? ____________________________________________
  9. On the remote server change directories to the Labs directory using the command cd Labs
  10. Create a new directory on the remote computer by using the command mkdir vilabs
  11. On the remote server change directories to the vilabs directory using the command cd vilabs
  12. On the local computer change directories to the TEMP directory by using the command lcd TEMP
  13. The file we are transfering is a compressed archive file and not just text. We should transfer this file using Binary mode.
  14. The default transfer mode is ASCII. Switch to Binary by using the command binary
  15. Transfer the file to the remote server by using the command put vilabs.tar.gz
  16. What transfer mode was used to transfer the file? ________________________________________________
  17. How would you verify that the file had been transferred? __________________________________________
  18. Close the connection by using the command close
  19. Exit the FTP client by using the command quit
You have now successfully transfered a file to the matrix server.

Open a terminal window connection to matrix and login using your account
Locate the file you just transfered. Change to the directory where it is stored.
We are going to extract the files from the compressed archive using the tar utility. (More on this utility later)
Execute the following command: tar xvzf vilabs.tar.gz

What files were extracted from the archive?






Remove the archive file by executing the command rm vilabs.tar.gz
Now we can practise transfering files fom matrix to our local hard drive.
  1. Open an FTP connection to matrix again and change to the correct directories.
  2. The files we are transfering now are ASCII text files. So we should make sure we transfer them in ASCII mode by using the command ascii>
  3. This time instead of "putting" the files on the server we are "getting" the files from the server.
  4. Enter the command get 1basics. How many files were transfered? _______________
  5. If we want to transfer multiple files at once we need to use command mget * This will transfer all files from the current remote directory.
  6. You will need to confirm the transfer of each file by typing y enter.
  7. Try transfering multiple files to matrix using the mput command.
  8. Close the connection and quit ftp
Command line FTP is a very handy skill to have and it is worth practising occasionally!!
